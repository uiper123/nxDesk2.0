# AI Roadmap — TTGTiSO-Desk (nxDesk 2.0)

Автономный план улучшений. Статусы: `[ ]` — в очереди, `[~]` — в работе, `[x]` — готово.

## Технический долг

- [x] **T1. Реальная тестовая инфраструктура фронтенда.** Установить Vitest + Testing Library + jsdom, заменить фейковые `*.test.tsx`-заглушки на настоящие тесты компонентов (Login, Logs, Settings, HostList). Добавить `npm test` в CI-цепочку.
- [x] **T2. Рефакторинг API-клиента (`services/api.ts`).** Таймауты через AbortController, типизированный класс `ApiError` с телом ответа сервера, конфигурируемый базовый URL, единая точка обработки сетевых ошибок.
- [x] **T3. Чистка Rust warnings.** Убрать dead_code в `api-server` (HostStatus::Busy и др.), прогнать `cargo clippy`, привести workspace к сборке без предупреждений.
- [x] **T4. Убрать мусор из фронтенда.** Удалить `console.log`/`console.error` из production-кода (заменить на централизованный logger), убрать `alert()` из Settings.

## UI/UX (премиальный уровень)

- [x] **U1. Дизайн-система.** Вынести цвета/тени/радиусы/типографику в CSS-переменные (`tokens.css`), единые компоненты состояний (loading skeleton, empty state, error state).
- [x] **U2. Toast-уведомления.** Глобальная toast-система (успех/ошибка/инфо) с анимациями вместо `alert()` и молчаливых ошибок.
- [x] **U3. Микро-анимации и полировка.** Плавные переходы между вью, hover-эффекты карточек хостов, анимированные статус-индикаторы (pulse для online), skeleton-загрузка списков.
- [x] **U4. Адаптивность.** Корректная работа при узких окнах: сворачиваемый сайдбар, грид хостов с auto-fit.

## Новые фичи

- [x] **F1. Dashboard (обзорная панель).** Стартовый экран после логина: счётчики хостов online/offline, активные сессии, последние события аудита, быстрые действия. Live-обновление.
- [x] **F2. Поиск и фильтрация хостов.** Поиск по имени/IP, фильтр по статусу, сортировка.
- [x] **F3. Health-индикатор API.** Статус соединения с api-server в сайдбаре (online/offline, latency) с автопроверкой.
- [x] **F4. Экспорт логов аудита.** Кнопка выгрузки журнала в JSON/CSV.

## Волна 2 — Сессия удалённого доступа (2026-06-10)

- [x] **S1. Командный центр сессии.** Overlay-панель «Session info» поверх VNC-канваса: статус туннеля, uptime, WebSocket/Upload endpoints, быстрый Reconnect. Health-классификация состояния соединения (`remoteAccess.ts` + юнит-тесты).
- [x] **S2. Режимы подключения.** Performance / Balanced / Clarity с рекомендованным масштабом и переключателем в тулбаре; смена масштаба больше не пересоздаёт VNC-подключение (исправлен баг с зависимостями эффекта).
- [x] **S3. Реальная телеметрия канала.** Инструментированный WebSocket (`vncChannel.ts`): фактический битрейт, частота кадровых обновлений и суммарный трафик вместо рандомных чисел; индикатор качества линка по реальным данным.
- [x] **S4. Честный pre-flight в ConnectionCard.** Вместо фейкового прогресса — реальные проверки: health API, наличие/статус хоста в реестре, поиск пользовательской сессии; отображение ошибок с Retry.

## Волна 3 — Ядро удалённого доступа (2026-06-12)

- [x] **C1. Договор возможностей (capability negotiation).** Новый модуль `protocol::capabilities`: токены `video.h264`, `capture.hardware`, `display.multi`, `session.resume`, `access.unattended` и др. `negotiate()` берёт пересечение клиент↔сервер в порядке предпочтения сервера, неизвестные токены игнорируются → старые и новые узлы совместимы.
- [x] **C2. Дельта-кодек экрана (tile-delta).** `video-pipeline::delta::DeltaEncoder` режет кадр на тайлы 64px, хэширует их (FNV-1a) и отправляет только изменившиеся; периодические keyframe'ы + keyframe при ресайзе/большом изменении. Заменяет «полный PNG каждый кадр» → радикально меньше трафика на статичном экране. Канал 3 в протоколе TTMP; TS-декодер `deltaDecoder.ts` рисует тайлы на `<canvas>`.
- [x] **C3. Мульти-монитор.** `video-pipeline::monitors` перечисляет физические мониторы (X11 RANDR / Win32 `EnumDisplayMonitors`, fallback — один синтетический). Протокол: `ListMonitorsRequest`/`MonitorList`/`SelectMonitor`; команда агента `monitors`; клиент `parseMonitorList`/`describeMonitor` + параметр `monitor` в WS-URL.
- [x] **C4. Непрерывность сессии (resume).** `session-manager::continuity`: при обрыве сети сессия «замораживается» с resume-токеном на grace-период (по умолчанию 60с) вместо немедленного сноса; повторное подключение по токену восстанавливает дисплей. Протокол: `ResumeSessionRequest`/`ResumeSessionResponse`, поле `resume_token` в `SessionStarted`.
- [x] **C5. Разрешения доступа (unattended vs «спросить пользователя»).** `security::access::AccessPolicy`: режимы `Unattended` (постоянный доступ по политике) и `AskUser` (локальное подтверждение с таймаутом), белый список пользователей, явный запрет приоритетнее. Протокол: `AccessRequest`/`AccessDecision`; клиент `getAccessModeDetails`.
