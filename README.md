# TTGTiSO-Desk 🖥️🔐

**TTGTiSO-Desk** — это современная кроссплатформенная система защищенного удаленного графического доступа для изолированных локальных сетей. 

Проект разработан с особым упором на соответствие требованиям безопасности **Astra Linux Special Edition 1.8 "Воронеж" (X11/Fly)**, обеспечивая строгую многопользовательскую изоляцию, шифрование трафика и высокую производительность видеопотока на базе GStreamer. Также полностью поддерживаются **Alt Linux**, **Arch Linux**, **Debian/Ubuntu** и **Windows**.

---

## 🛠️ Основные возможности

* **Высокая производительность:** Динамическое сжатие кадров и кодирование H.264 с использованием GStreamer.
* **Строгая безопасность:** Аутентификация по ключам Ed25519, встроенный модуль аудита безопасности событий (вход, передача файлов, буфер обмена).
* **Интегрированные функции:** Буфер обмена, передача файлов, автообнаружение хостов в локальной сети по UDP.
* **Многопользовательская изоляция:** На Linux-серверах сессии запускаются в изолированных X-серверах (Xvfb/Openbox).
* **Кроссплатформенность:** Клиент работает на Linux и Windows. Серверный агент запускается как демон `systemd` на Linux и как служба `LocalSystem` на Windows.

## 🔑 Как работает авторизация по SSH-ключам

Для подключения к серверному агенту используется авторизация по паре SSH-ключей (Ed25519). 
1. **Генерация:** Клиентское приложение генерирует пару ключей автоматически при первом запуске (или в настройках).
2. **Публичный ключ:** Этот ключ (например, `ssh-ed25519 AAAAC3... client`) прописывается на удаленном хосте в файле `authorized_keys`.
3. **Бессрочность:** Сгенерированный ключ является **постоянным (бесконечным)**. Его не нужно менять после перезапуска программы, сервера или смены портов. Обновление ключа требуется только при ручном отзыве доступа или переустановке клиента с удалением конфигурации.

## 🚀 Установка серверного агента (Server Agent)

Серверный агент должен быть установлен на компьютере, к которому планируется подключаться удаленно.

### 🐧 Автоматическая установка на Linux (Astra, Alt, Arch, Ubuntu, Debian, RedHat)

Мы подготовили универсальный скрипт автоматической установки, который сам определит вашу ОС, установит нужные зависимости, настроит стандартный SSH-сервер (порт 22) и пропишет публичный ключ авторизации.

1. Скопируйте публичный ключ вашего клиента.
2. Запустите скрипт установки, передав ключ в качестве аргумента:

```bash
sudo ./scripts/install-agent.sh "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIDJrmSItu4mURAJadyzxWTZJtN3CVEQB6LUpHb9LTRJY TTGTiSO-Desk client"
```

**Что сделает скрипт:**
* Установит системные пакеты (SSH-сервер, `xvfb`, `openbox`, `xauth`, `socat`, `openbsd-netcat`, плагины GStreamer).
* Запустит и включит автозапуск SSH-демона на порту 22.
* Пропишет переданный ключ в `~/.ssh/authorized_keys` пользователя, выполнившего установку, с безопасными правами доступа.
* Скопирует бинарный файл агента, настроит его порт по умолчанию (`2222`).
* Установит и запустит службу `systemd` (`ttgtiso-desk-agent.service`).

### 🏁 Автоматическая установка на Windows

На Windows агент работает как фоновая служба Windows, не требуя сторонних графических серверов.

1. Откройте консоль **PowerShell от имени Администратора**.
2. Перейдите в папку со скриптами и запустите:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\install-agent.ps1 -SshKey "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIDJrmSItu4mURAJadyzxWTZJtN3CVEQB6LUpHb9LTRJY TTGTiSO-Desk client"
```

**Что сделает скрипт:**
* Проверит наличие встроенного в Windows **OpenSSH Server**. При отсутствии установит его через Windows Capabilities.
* Запустит службу Windows `sshd` и настроит автозапуск на порту 22.
* Пропишет публичный ключ в профиль текущего пользователя (`~/.ssh/authorized_keys`) и в файл глобальных администраторов (`C:\ProgramData\ssh\administrators_authorized_keys`) с безопасными правами доступа (`icacls`).
* Установит службу Windows `TTGTiSODeskAgent` (бинарный файл устанавливается в `Program Files`, конфигурационный файл — в `ProgramData\TTGTiSO-Desk\agent.toml`).

## 💻 Установка клиентского приложения (Desktop Client)

Клиентское приложение используется администратором для управления хостами и непосредственного подключения.

### Для Arch Linux / Manjaro (Рекомендуемый)
Установка нативного пакета полностью решает проблему совместимости с драйверами Mesa/EGL (исключает ошибку `EGL_BAD_PARAMETER`).

**Вариант 1: Сборка из исходников с помощью `makepkg`**
```bash
git clone https://github.com/uiper123/nxDesk2.0.git
cd nxDesk2.0
makepkg -si
```

**Вариант 2: Установка готового пакета из релизов**
Скачайте актуальный файл `*.pkg.tar.zst` со страницы релизов GitHub и установите его:
```bash
sudo pacman -U ttgtiso-desk-0.1.14-1-x86_64.pkg.tar.zst
```

### Для Windows / Debian / Ubuntu / Astra
* Скачайте готовый установщик со страницы релизов GitHub:
  * **Windows:** `.msi` или `.exe`
  * **Astra / Debian / Ubuntu:** `.deb` пакет
  * **RedHat / CentOS:** `.rpm` пакет

## 🔧 Инструкции для разработчиков

### Требования для сборки
* **Rust** 1.77+
* **Node.js** 20+ / **NPM** 10+
* **Системные библиотеки (Linux):** `libgstreamer1.0-dev`, `libgstreamer-plugins-base1.0-dev`, `libwebkit2gtk-4.1-dev`, `libappindicator3-dev`, `librsvg2-dev`, `patchelf`.

### Быстрый запуск окружения разработки
В корне проекта находится скрипт `./dev-start.sh`, который автоматически запускает локальный агент, API-сервер и десктопный клиент Tauri в режиме отладки:
```bash
./dev-start.sh
```

### Сборка вручную
* **Сборка Rust бэкенда:**
  ```bash
  cargo build --release
  ```
* **Запуск тестов:**
  ```bash
  cargo test --workspace
  ```
* **Запуск React фронтенда клиента (Vite):**
  ```bash
  cd apps/desktop-client
  npm install
  npm run dev
  ```

### Выпуск нового релиза (CI/CD)
Проект настроен на сборку релизов через GitHub Actions. При изменении версии в `tauri.conf.json` и пуше тега автоматически собираются все дистрибутивы.

Для обновления версии используйте скрипт:
```bash
./scripts/bump-version.sh 0.1.14
git push origin main --tags
```

## 🔍 Решение проблем (Troubleshooting)

#### 1. Ошибка запуска GUI на Arch Linux (AppImage)
Если AppImage падает с ошибкой инициализации EGL, используйте установку нативного пакета `.pkg.tar.zst` или соберите его через `makepkg -si`. Это свяжет Tauri с системными библиотеками графического драйвера напрямую.

#### 2. Агент недоступен после установки
* Убедитесь, что служба запущена:
  * **Linux:** `sudo systemctl status ttgtiso-desk-agent`
  * **Windows:** `Get-Service TTGTiSODeskAgent`
* Проверьте, что SSH-сервер работает на порту 22: `ssh localhost` или `ssh-keyscan -p 22 localhost`.
* Убедитесь, что брандмауэр/Firewall разрешает входящие соединения на порт 22 и порт 2222 (порт работы агента).
